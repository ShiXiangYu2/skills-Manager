# 写作复盘

## 复盘流程

```
评分结果 → 分析经验教训 → 更新技法库评分 → 记录复盘结果
```

## 运行复盘

```bash
# 对章节进行复盘
curl -X POST http://127.0.0.1:8001/api/chapters/{id}/retrospective
```

## 复盘结果示例

```json
{
  "chapter_id": 83,
  "lessons_learned": "开篇不能只做场景铺垫，必须在200字内引入核心人物或冲突",
  "updated_techniques": ["悬念前置"],
  "patterns_to_avoid": [
    "纯环境描写超过200字而不引入人物或冲突",
    "结尾缺乏钩子，没有制造立即的阅读期待"
  ]
}
```

## 复盘输出

每次复盘会输出：

1. **经验教训**：本轮学到什么
2. **技法反馈**：哪些技法好用/不好用
3. **技法更新**：更新技法库的 quality_score
4. **避免模式**：以后要避免什么
5. **新发现**：有没有发现新的可复用技法

## 技法库更新

复盘会自动更新技法库：
- 使用效果好的技法 → quality_score +0.5
- 使用效果差的技法 → quality_score -0.5
- 连续 3 次效果好 → 标记为"核心技法"
- 连续 3 次效果差 → 标记为"待修正"

## 闭环

```
读书笔记 → 蒸馏技法 → 写作 → 评分 → 复盘 → 更新技法 → 下一章更强
```

这就是"越写越强"的核心机制。
